Strategic Context

Five Strategic Meta-Narratives

Before the cluster-by-cluster breakdown, five overarching friction patterns emerge from the full pipeline. These are not individual features — they are the systemic gaps that define how clients experience the ceiling of the platform.

🏔️
The Power User Ceiling
Clusters 1, 3, 4 · Marktplaats, Kleinanzeigen, Edeka, CTS Eventim

The platform is built for marketers, but data engineers keep showing up. Every mature client reaches a moment where the UI is not enough — they need SQL for calculated attributes, API access to programmatic exports, and operator logic that matches their specific data model. The SQL CA cluster (157 tickets), the operator cluster, and the export cluster all trace back to the same root: the abstraction layer is optimised for simplicity, but complex clients want to reach through it.

🕒
Confidence in Data Freshness
Clusters 2, 8 · Edeka, Sky, REWE, Kleinanzeigen

A disproportionate share of tickets ask one question: when exactly was my data last updated, and can I control when it refreshes? Audience refresh scheduling, ingestion monitoring, and alerts are all the same enterprise anxiety in different clothes. Clients have downstream SLAs and media campaigns that depend on data being fresh before they act. The platform confirms ingestion happened — but not with enough granularity for enterprise operations.

⚡
Always-On Automation
Clusters 6, 7 · Sky IT, O2, Mediamarkt

Clients want journeys that react to what customers are actually doing — in real time. The gap between a scheduled journey and an event-triggered journey is the gap between batch marketing and real-time personalisation. With only 19% GA, Cluster 7 is the most under-delivered major theme in the pipeline. Sky IT Copernico is at churn risk because of this gap.

📉
The Unmeasured Destination
Cluster 3 · 74 tickets, 34% GA, 14 client mentions

Audiences are sent to destinations, but clients cannot see what happened next. Match rates, send counts, export logs — Cluster 3 is almost entirely about visibility into activation outcomes. Clients cannot optimise what they cannot measure. Many have resorted to manual spreadsheet exports and Python scripts querying destination platforms directly as workarounds.

🔐
RBAC Too Coarse
Cluster 9 · 42% rejection rate — highest of any cluster

The permission model works for small teams but not for enterprise multi-market deployments. Every enterprise client with multi-team or multi-market operations hits the RBAC ceiling. The 42% rejection rate has not reduced demand — it has grown. This is now a documented pre-sales obstacle in financial services, retail media, and broadcasting procurement assessments.
